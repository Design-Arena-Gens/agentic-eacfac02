# SensorDashboard (GTK# + OxyPlot)

Desktop панель мониторинга для Linux, написанная на .NET 8 с использованием GtkSharp и OxyPlot. Приложение отображает состояние производственных датчиков: список слева, три вкладки с графиками по центру и двойную аналитическую панель снизу.

## Возможности

- Верхнее меню с командами обновления и ручного шага симуляции
- Панель датчиков слева с цветовой индикацией статуса и детальной карточкой
- Три вкладки с временными графиками (температура, влажность, давление) на базе OxyPlot
- Нижняя двойная панель: агрегированные значения и список текущих предупреждений
- Встроенный симулятор, обновляющий показатели каждые 4 секунды

## Требования

- .NET SDK 8.0+
- GTK3 runtime (`libgtk-3-0` и зависимости)

Для Debian/Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y libgtk-3-0
```

## Запуск

```bash
cd SensorDashboard
dotnet restore
dotnet build
dotnet run
```

> Примечание: для запуска требуется графическая сессия X11/Wayland. В headless-средах сначала поднимите виртуальный дисплей (`xvfb-run dotnet run`).

## Структура

```
SensorDashboard/
├── Program.cs              // Точка входа и инициализация GTK
├── MainWindow.cs           // Построение интерфейса и логика обновления
├── SkiaPlotView.cs         // Обёртка для экспорта OxyPlot в Gtk.Image
├── SensorRepository.cs     // Симулятор и хранилище значений датчиков
├── Sensor.cs               // Модель датчика
├── SensorReading.cs        // Запись временного ряда
└── SensorStatus.cs         // Перечисление статусов
```

## Тестирование

Проект собирается командой `dotnet build`. Автотестов нет, так как основной функционал связан с GUI. Для проверки визуальной части выполните `dotnet run` в окружении с доступным дисплеем.
